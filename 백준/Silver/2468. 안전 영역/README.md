# 04/03/25 TIL 

### 🧠 문제 요약
- 지역의 높이 정보를 바탕으로 **비가 내렸을 때 물에 잠기지 않는 안전한 영역의 최대 개수**를 구하는 문제.
- 비의 양을 0부터 최고 높이까지 하나씩 늘리며 안전 영역 개수를 탐색.
- 안전 영역은 **상하좌우로 연결된 높이가 비의 양보다 높은 지점들**로 구성됨.

---

### 🔧 사용한 기술 및 접근 방법
- **DFS(깊이 우선 탐색)**: 연결된 안전 영역을 탐색하는 데 사용.
- **Brute Force**: 모든 강수량(높이) 경우에 대해 완전탐색.
- **그래프 탐색 문제**로 분류 가능.

---

### 📌 핵심 아이디어
1. 주어진 지역에서 최대 높이(`top`)를 찾음.
2. 강수량 `t`를 0부터 `top-1`까지 하나씩 높여가며:
   - 각 높이에 대해 **방문하지 않았고 물에 잠기지 않은 위치**에서 DFS 실행.
   - DFS로 한 영역을 모두 방문 처리.
   - DFS 호출 횟수를 영역 개수로 카운팅.
3. 각 강수량별 영역 개수 중 최대값을 기록.

---

### 📈 성능 요약
- **메모리**: 349,720 KB  
- **시간**: 748 ms  
- **문제 분류**: DFS, 완전탐색, 그래프 탐색  

---

### 📝 배운 점
- **탐색 문제에서 visited 초기화를 반복 루프마다 수행해야 정확한 영역 계산이 가능**하다는 점을 숙지.
- `sys.setrecursionlimit`을 통해 깊은 DFS 호출 대비 가능.
- 불필요한 DFS 호출을 막기 위한 `visited` 체크가 중요.
- 완전탐색도 충분히 최적화 가능하면 100×100 사이즈 문제 해결 가능.
